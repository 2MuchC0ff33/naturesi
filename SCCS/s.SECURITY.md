# Security Policy

We take security seriously. If you discover a vulnerability in this project, please do NOT open a public issue with sensitive information.

Preferred reporting options

- Use GitHub Security Advisories (recommended) to disclose vulnerabilities privately and coordinate fixes.
- If you cannot use GitHub Security Advisories, open a private (non-public) issue and mark it clearly as a security report so maintainers can triage it. Avoid posting any secret data or exploit details in public channels.

What to include

- Affected component or file(s)
- Steps to reproduce (minimal) or a proof-of-concept (attach privately)
- Expected and actual behaviour
- Version or commit hash where the issue was observed

Response policy

Maintainers will acknowledge reports within a reasonable timeframe and will coordinate fixes and disclosure. We follow responsible disclosure practices and will work with reporters to release fixes before public disclosure where practical.

Privacy

This repository is privacy-first. Do not include personal data or secrets when reporting issues. If we need additional private information to investigate, maintainers will request it through a private channel.
